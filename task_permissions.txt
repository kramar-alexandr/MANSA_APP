ВОЗМОЖНЫЕ ПОЛЯ ДЛЯ УРОВНЕЙ ДОСТУПА
  Для модулей: "SHOW"
  Для регистров и настроек: "SHOW", "READ", "LIST", "EDIT", "CREATE", "REMOVE", "COPY"
  Для полей: "READ","EDIT","CREATE"
  
ОСОБЕННОСТИ:
  -отсутствие "ALL","NONE"
  -отсутствие уровней доступа как таковых для модулей (только управление отображением)
  
  
ОБЪЯСНЕНИЯ:
"SHOW" - управляет только визуальным отображением модуля (регистра, настройки) в списке модулей (регистров, настроек).
        Если не указан - скрывать модуль(регистр.)
        "modules" : {
            "defaultAccess": ["SHOW"],
            "Test":{
              "access":[]
            }
        }
        В примере будут отображены все модули, за исключением "Test"

"READ" - пользователь имеет право просматривать существующ. регистр или поле.

"LIST" - пользователь имеет право просматривать весь список всех записей отдельного регистра.

"EDIT" - пользователь имеет право редактировать существующие записи или поле.

"CREATE" - пользователь имеет право создавать новые записи. Для поля это означает возможность ввести
          новые значение в пустое поле при редактирование записи.

"REMOVE" - пользователь имеет право удалять существующие записи

"COPY" - пользователь имеет право дублировать существующие записи
